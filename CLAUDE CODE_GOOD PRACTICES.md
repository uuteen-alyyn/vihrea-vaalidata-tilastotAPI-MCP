# Good practices

## Implementation_plan.md
All projects should include a clear, step-by-step implementation plan, where progress is planed in a clear to do list phasion. The project is split into individual phases. Each phase has goals and tests

## Logbook.md
All projects should have a logbook.md, where all changes and action are documented so that the newest entry is always at the bottom. The coding agent is **never** allowed to remove anything from the file, only add new entries to the bottom.

--- example entry ---
## ENTRY DESCRIPTIVE TITLE YYYY-MM-DD HH:MM:SS
[what was done - short description]
[possible made decissions]
[test results]
[any notes]
--- / example entry ---
